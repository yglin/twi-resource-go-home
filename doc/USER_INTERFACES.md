# 資源勾引魟 (ResourceGoingHome) - 實作前端 UI 介面文件 (USER_INTERFACES.md)

本文件完整記錄「資源勾引魟」Web 應用程式目前已實作的系統路由架構、各頁面詳細 UI 設計 layout、核心互動式 UI 元件，以及系統整體配色與字型主題規範。

---

## 一、 全域系統路由與頁面架構 (System Routes Architecture)

本應用程式基於 React Router (v6) 與守衛機制（ProtectedRoute）實現了嚴格的雙重角色存取權益劃分：

| 路由路徑 (Route Path) | 權限要求 (Access Guard) | 頁面元件 (Page Component) | 核心功能概述 |
| :--- | :--- | :--- | :--- |
| **`/`** | 公開 | `LandingPage` | 永續海洋風格入口網、宣傳看板、直覺式登入引導。 |
| **`/auth`** | 公開 | `AuthPage` | Google、Email 快速註冊與登入入口切換。 |
| **`/unauthorized`** | 公開 | `Unauthorized` | 權限不足引導頁、提供返回個人工作區的快捷入口。 |
| **`/setup`** | 登入者 | `ProfileSetup` | **關鍵首航引導頁**：多重角色自選、地理座標、梅克魚時段增添、勾引魟材質與專屬處理指引設定。 |
| **`/maker`** | 梅克魚限制 | `MakerDashboard` | 梅克魚工作區首頁：列表展示與生命週期狀態 Badge。 |
| **`/maker/new`** | 梅克魚限制 | `CreateRecord` | 快捷 AI 辨識相機上傳頁面、二進位影像分析、自適應材質新增。 |
| **`/maker/record/:id`** | 梅克魚限制 | `RecordDetails` | 物資歷程狀態牆：查看推薦魟魚名單、地圖定位校正與一鍵向魟魚發送委託。 |
| **`/maker/notifications`** | 梅克魚限制 | `Notifications` | 歷史任務通知流：提醒梅克魚完成前置整理、通知確認上車與收取完畢。 |
| **`/going-home`** | 勾引魟限制 | `GoingHomeDashboard` | 勾引魟工作區（等同 `/going-home/requests`）：可用收運請求工作面板。 |
| **`/going-home/requests`**| 勾引魟限制 | `AvailableRequests` | 指名委託魟魚請求列表：詳細地址地圖展示、一鍵同意收運、退單並回填不便收取理由。 |
| **`/going-home/plan`** | 勾引魟限制 | `ActivePlan` | 點對點智慧航線執行面板：AI/演算法路徑拓撲推薦、一鍵核准、當場已上車 / 跳過節點、一鍵送抵處理站。 |
| **`/going-home/profile`**| 勾引魟限制 | `RayProfile` | 勾引魟專屬回收物品與專屬整理期望微調、地址更改設定。 |
| **`/recycleContract`** | 登入者 | `ContractDashboard` | 回收契約列表：按參與角色（梅克魚/勾引魟/瑞莎魺）分類展示契約卡片、建立契約入口（限勾引魟）。 |
| **`/newRecycleContract`** | 勾引魟限制 | `NewRecycleContract` | 建立或微調編輯回收契約：包含回收記錄範本、排程控制、與瑞莎魺相容性下拉篩選。 |
| **`/recycleContract/:id`** | 契約成員限制 | `ContractDetails` | 三方簽署審批控制台、高密度歷史歷程軌跡板、留言板交流區、暫停與重啟操作按鈕。 |
| **`/openForAll`** | 公開 | `OpenForAll` | **公開徵收物資專區**：列出所有狀態為「公開徵收 (OPEN_FOR_ALL)」的回收物資，所有使用者（包含訪客及登入者）皆可瀏覽，已授權之勾引魟可在此進行接單。 |
| **`/admin`** | 管理員限制 | `AdminDashboard` | 系統主資料管理後台：統計看版與 `masterData_resources` 的 CRUD 操作。 |

## 二、 核心工作區佈局與導覽設計 (Workspace Layout)

系統針對「梅克魚」與「勾引魟」兩個身分設計了具備強烈品牌調性的雙重邊欄。

### 1. 梅克魚工作區側邊欄 (Maker Sidebar)
* **配色風格：** 採用高質感冷色調深石炭灰 (`bg-slate-900`)，搭配海洋青藍 (`text-cyan-400`/`bg-cyan-500`) 做為主要亮色與陰影點綴。
* **雙重角色無痛轉換開關：**
  - 當登入使用者同時被授權具有雙身分時，側邊欄頂部 Logo 旁會渲染出藍色小標記。
  - 點擊「梅克魚空間」會觸發縮放與微形彈跳動畫，平滑切換至「勾引魟工作區 (`/going-home`)」。
* **公開徵收分頁連結：** 側邊欄配置「公開徵收」地球圖示導覽項目，點擊即可無縫跳轉至 `/openForAll` 頁面。
* **快捷功能懸浮鈕 (FAB)：** 右下角恆常跟隨一個大型海洋青藍色 `+` 按鈕，支持在任何梅克魚子頁面一鍵點擊，拉起 AI 相機上傳。

### 2. 勾引魟工作區側邊欄 (Going Home Sidebar)
* **配色風格：** 同樣採用 `bg-slate-900` 尊榮深黑，但輔以 **陸地永續靛藍 (`text-blue-400`/`bg-blue-600`)** 作為視覺主題，以在心智模型中將其與「物流/計畫/收運」完美扣合。
* **頂部角色瞬轉開關：** 點擊會流暢滑向「梅克魚空間 (`/maker`)」。
* **公開徵收分頁連結：** 側邊欄配置「公開徵收」地球圖示導覽項目，點擊即可跳轉至 `/openForAll` 頁面以查看可接之公開案件。

## 三、 頁面細部 UI 設計 (Page Layout Details)

### 1. 入口宣傳頁 (`LandingPage`)
* **沉浸式水底光流背景：** 使用具備空氣感的淡雅波波白 (`bg-slate-50`)與透光漸變背景，渲染出乾淨的呼吸感，排除了低質感 AI 雜亂感。
* **公開徵收市場快捷紐：** 右上角導覽欄包含「公開徵收市場」快捷按鈕，允許訪客及使用者在登入前一鍵進入公開徵收資材市場查看全台資材。
* **特色卡片區：** 提供「雙邊參與，漁魟共生」的主視覺卡片，帶領用戶迅速理解梅克魚、勾引魟在系統中各司其職的趣味分工。

### 2. 引導補全頁 (`ProfileSetup`)
* **漸進式卡片佈局：** 將多步驟引導整合於乾淨的大型卡片中。
* **角色卡片切換 (Onboarding CardPicker)：**
  - 水平並列兩張「梅克魚」與「勾引魟」插圖角色大按鈕。
  - 觸發選取後帶有精美、柔和的海洋波紋外框高亮邊界。
* **動態指引編輯板：**
  - 當用戶選取右側「勾引魟」，下方會拉起他所複選的所有可回收物品 (EX: 塑膠寶特瓶、瓦楞紙箱)。
  - **自動化行同步：** 每多勾一類，卡片下方就自動彈射出一行「該類別的專屬清理期待叮嚀」輸入框。

### 3. AI 影像拍照與自適應檢視頁 (`CreateRecord`)
* **二進位極敏拖曳與相機熱點區：**
  - 具備實線感與虛線交錯的相機圖標上傳。點擊即可在流動裝置上直接呼叫自備相機。
* **海洋氣泡波紋載入骨架 (Ripple Skeleton Loader)：**
  - 當二進位 Base64 數據送往 Gemini 時，系統遮罩一層動態的水波載入特效，徹底舒緩了伺服器辨識時使用者的等待焦慮。
* **自適應表單覆寫：**
  - AI 辨識完成後自動將預填資料流暢注入文字输入框，包括材質、名稱、預估數量以及自動預填/學習得到的「計量單位」（比如「瓶」、「公升」、「片」或預設「個」）。
  - 利用不同透明度高亮提醒哪些是 AI 填寫、哪些可以用戶覆寫。使用者可以自由修訂數值的計量單位以符合真實物資情形。
* **預估收購價格動態估值卡片 (Estimated Price Indicator Card)**：
  - 在數量與分類表單下方，特製一個淡鵝黃色背景的高質感「預估收購價格 (NTD)」資訊區塊。
  - 當材質、產品分類與數量（`quantity`）變更時，介面會實時根據主檔對應品類的 `avgPrice` 與 `estimatedWeight` 進行累乘計算並渲染出動態價格（如 `15 元`）。
  - 右方帶有圓形琥珀色警告 `!` 按鈕，點擊能拉起 Dialog 說明此估值並非最終收購價格，若主檔資料不足（或為0）則保底顯示為 0 元。

### 4. 智慧航線調度與出發計畫面板 (`ActivePlan`)
* **進度路線拓撲軌跡 (Route Topology Stepper)：**
  - 使用垂直精細的虛實線點狀插圖（Stepper Line）呈現本次要去的所有停靠站。
  - **即時按鈕反饋**：每個站點卡片下方配有簡潔的 `🚚 確認點收上車` 與 `異常跳過` 操作，支持單站點獨立前進並即時發送綠色推播。
* **模擬物流地圖 (Interactive Polyline Map Box)：**
  - 計劃內包含 AI 依承載體積智慧預測出來的「最省碳交通工具」（例如：低碳重卡、電動三輪）。地圖區域流暢繪製出停靠順序線路（拓撲折線）。

### 5. 回收契約主控儀表板 (`ContractDashboard`)
* **三方角色分頁視窗 (Category-Based Contract Tabs)：**
  - 設計有三個平行分頁（Tabs），將與當前登入者相關的契約依使用者於合約中的角色分類展出：
    1. **「我產出的契約 (資源梅克魚)」**
    2. **「我載運的契約 (資源勾引魟)」**
    3. **「我收購的契約 (資源瑞莎魺)」**
  - 使用者可以根據其角色在不同分頁流暢穿梭。
* **分區合約狀態卡片 (Split Status Grid) & 新增契約鈕：**
  - 每個分頁下，依合約狀態進行垂直卡片式排版。
  - 當且僅當當前登入使用者具有 **資源勾引魟** 權限時，介面右上角與畫面中空狀態時將高亮渲染出「新增契約 (Add Contract)」的海洋靛藍按鈕。

### 6. 建立/修改定期回收契約頁 (`NewRecycleContract`)
* **三段式卡片式表單 Layout (Form Section Cards)：**
  - **1. 回收紀錄範本區**：一鍵引用或手動鍵入。提供滑動式的數量阻尼計量器與品類下拉選單。若是由已完成的實體收運單跳轉而來，此區卡片將伴隨精美閃動特效，無縫預載所有預設資材與預計數量，完美防呆。
  - **2. 排程控制設定區**：直覺的時間選擇器 (HH:MM) 搭配單選按鈕（每日 / 每週 / 每月）。當按下「每週」時，彈射出七天 (0-6) 選擇器；當按下「每月」時，展開 1-31 日精緻網格，幫助用戶直覺式排程。
  - **3. 聯袂三方角色綁定區**：
    * 資源梅克魚：自動渲染系統中所有登入之梅克魚角色。
    * 資源勾引魟：系統唯讀鎖死，直接顯示當前登入魟魚名稱及精緻頭像。
    * 資源瑞莎魺：**「自適應相容指引篩選器」**。當用戶在上端設定完物資類別，系統將會在後台利用交叉聯集，動態篩選出回收項目包含此物資的「資源瑞莎魺」，保障派單不衝突。不符指引之瑞莎魺在下拉選單中將直接灰化，並顯示警示小字（例：『瑞莎阿明未收購此項目』）。

### 7. 契約簽署與生命週期控制台 (`ContractDetails`)
* **三方聯手卡片式簽章卡 (Triple Status Checker)：**
  - 水平排列三張簽字卡（梅克魚、勾引魟、瑞莎魺）。
  - 對於當事人之簽署，系統高亮「簽核同意」與「退回拒絕」雙按鈕：
    * 點擊同意時，彈出「客製化角色自律誓約二次確認」彈窗（包含其角色應遵從的配送與秤重時限警語）。
    * 點擊拒絕時，彈出「反饋與退件對話框 (Rejection Dialogue)」，要求強制填入具體退件理由。

### 8. 系統管理工作台與使用者管理控制台 (`AdminDashboard`)
* **多維度管理工作艙 (Admin Workspace Hub)：**
  - 左側採用獨立垂直 Sidebar，提供「資源主檔管理」、「建議資材審核」、「系統運作日誌」與「使用者管理」四大核心功能分頁，按鍵帶有平滑流暢的縮放高亮微動效。
* **系統使用者管理面板 (User Management Desk)：**
  - **使用者資料表格 (Users Grid List)**：直覺化大卡片表格，呈現使用者頭像（若無則顯示首字字母組成的氣泡圖標）、顯示名稱、Email與電話、所持有的複合身份角色 Badge 陣列（梅克魚 🐟、勾引魟 ✈️、資源瑞莎魺 🪸、管理員 ⚙️）以及登記通訊地址。
  - **三維度複合過濾條 (Tri-Dimension Query Bar)**：
    1. *模糊搜尋框*：對姓名、Email、聯絡電話、詳細地址進行全欄位實時模糊匹配。
    2. *角色篩選下拉選單*：快速按單一特定角色進行帳號分流。
    3. *收受資材品類下拉篩選器*：**業界首創雙維交叉過濾器**。自動加載材質主檔 `masterData_resources` 中的品類（格式：`📦 [材質] - [產品]`）。管理員選定後，系統將實時比對各個用戶的 `acceptedCategories`（收購大類）與 `recoveryGuides`（回收指引），瞬時過濾出提供該特定回收服務的使用者。
  - **使用者詳細資料檢視彈窗 (Detailed User Profile Modal)**：
    - 點擊表格行或 Eye 眼睛按鈕可拉起大尺寸沉浸式毛玻璃彈窗。
    - 完整呈列使用者檔案資料、時段配置及回收指南清單。
    - **定價單位一致性**：當展示資源瑞莎魺的收購報價指南時，估價值單位一律強制規格化渲染為 **`元 / 公斤`**，視覺上乾淨俐落，確保無歧義。
* **資源主檔管理面板 (Resource Catalog Panel)：**
  - **資料顯示表格 (Resources Table)**：表格中列出材質、產品分類、過期時數、**預估單件重量 (kg)**、預設減碳效益、平均收購價與預設建議，管理員可一眼掌握各資材的物理特徵（預估單件重量以高亮的藍色單色微點綴展示，如 `0.025 kg`）。
  - **資材主檔與建議編輯彈窗 (Resource & Suggestion Editor Dialogs)**：當點擊新增、編輯資源或編輯提報建議時，彈窗中配備 **「單件預估重量 (公斤/個)」** 輸入控制項（支持精確至小數點後三位的 `number` 類型 input 欄位，例如 `0.025` 公斤），允許管理員對該類別的單件物理預估重量進行動態錄入與修訂。
  - **同步計算平均收購價按鈕 (Sync average price button)**：主檔工具列高亮配備「同步計算平均收購價 (用戶端)」備援按鈕。點擊後會顯示輕量化載入狀態，在前端安全高效地批次調度並重新加權計算各資材的平均收購報價，並利用 `writeBatch` 更新回 Firestore，提供全天候的業務備援高可用性。

## 四、 核心客製化 UI 元件 (Interactive Design Systems)

我們大量避免使用未經微調的元件。系統採用 Tailwind CSS 精巧整合了以下高互動 UI 單元：

### 1. 狀態生命週期 Badge (StatusBadge)
根據狀態機，精緻映射不同的字體色、背景色與邊界色：
* **單次回收記錄狀態 (`RecordStatus`)**：
  - `JUST_BORN`: 軟萌嫩草綠 (`bg-emerald-50 text-emerald-700 border-emerald-200`) ── 象徵剛誕生的物資，充滿新生期盼。
  - `WAITING_FOR_COLLECTION`: 深曜海洋藍 (`bg-sky-50 text-sky-700 border-sky-200`) ── 等待點收承接。
  - `COLLECTION_CONFIRMED`: 暗夜尊爵紫 (`bg-violet-50 text-violet-700 border-violet-200`) ── 計畫已妥，安心排程。
  - `PICKED_UP`: 卡其物流橘 (`bg-amber-50 text-amber-700 border-amber-200`) ── 正在出發運送途中。
  - `COMPLETED`: 圓滿極光綠 (`bg-green-100 text-green-800 border-green-300`) ── 給予使用者最高光之回饋。
* **長期回收契約狀態 (`ContractStatus`)**：
  - `Pending Signatures`: 鵝黃審查橙 (`bg-amber-50 text-amber-700 border-amber-200`) ── 待三方共同審閱簽字。
  - `Active`: 湛藍運行洋 (`bg-blue-50 text-blue-700 border-blue-200`) ── 啟用定期排程、循環履約中。
  - `Rejected`: 珊瑚烈焰紅 (`bg-rose-50 text-rose-700 border-rose-200`) ── 因故退回拒絕，高亮標明退件理由。
  - `Suspended`: 煙雨寂境灰 (`bg-slate-100 text-slate-700 border-slate-300`) ── 臨時在線中斷暫停，排程生成器冬眠。

### 2. 無法收取退單提示組 (UnableCollectModal)
當勾引魟因不可抗因素取消該次特定託付時，為維持梅克魚與魟魚之間的誠信感，此提示組整合了：
* 嚴謹明晰的驚嘆號插圖外框。
* 特製的不便收取理由快速選單（提供：「交通工具載重限制」、「配合時段衝突」、「材料狀況不佳/未清洗」、「地址不在指定公里內」等多種理由快捷回填，或選擇鍵入自訂原因說明）。
* 取消釋放後，卡片帶有淡出（Fade-out）轉場動作。

### 3. 多時段自由新增列 (TimeWindowRepeater)
於補全資料與個人設定頁面中：
* 用戶可點擊「＋ 追加可用時段」。
* 彈出小型的星期天數選單（0-6 周日到周六）與兩端精確時間阻尼器。
* 每行配備微型垃圾桶圖標，點擊時伴隨彈跳，提供毫無延遲、無壓力的快感操作。

### 4. 契約高密度歷史稽核歷程牆 (ContractHistoryLogger)
* **視覺呈現 (High-Density Linear Timeline)**：
  - 垂直極窄的永續靛藍極細導引線，依降序（最新動作在上）排版。
  - 每個歷程圓點依據使用者角色顯示前綴 icon（梅克魚 🐟、勾引魟 ✈️、瑞莎魺 🪸、系統自動 ⚙️）。
  - 對於操作者之姓名、時戳 (由 JetBrains Mono 精確顯示至秒) 進行科學高對比渲染。
  - 若動作伴隨了具體附言（例如暫停理由：『這週我因出差無法打包...』），該段備註將以極致高雅的斜體灰字（`italic text-slate-500`）伴隨淡灰色底板區塊呈現，資訊完整且絕無繁雜。

### 5. 三方協同對話留言板 (ContractChatConsole)

## 五、 配色與字型主題規範 (Color & Typography Specs)

### 1. 配色系統 (Palette Specs)
* **海洋深邃黑 (`Slate-900`/`100`):** 應用系統的基礎框架邊欄色、與主標題高硬度色。
* **梅克魚主色 (`Cyan-500` / `#06b6d4`):** 代表海洋中最活躍透亮的青色，高顯色度，作為梅克魚專屬交互與主要懸浮鈕、進度條之高光配色。
* **勾引魟主色 (`Blue-600` / `#2563eb`):** 代表陸地實踐、引導船航的深翠靛藍，帶給大家安全、踏實、且信任的物流航線感。
* **背景基底調 (`Slate-50` / `#f8fafc`):** 去除粗糙的高對比亮白，預設以不刺眼且高雅呼吸感極佳的灰白作為襯底底稿。

### 2. 字型設定 (Typography Config)
在 `src/index.css` 與全系統中，精確綁定了字型意象：
* **無襯線主字型 (Inter)：** 全域 UI 框架、說明內文、按鈕面板與對話框，極低干擾性，可讀性高。
* **數據/計數/特定狀態 (JetBrains Mono)：** 所有的 `UpdatedAt` 時戳、經緯度坐標、垃圾計數量與 Geohash 哈希文字中，均指定以 code font 渲染，傳達絕對科學、精確與乾淨的主觀工業工藝感。
