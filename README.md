# 資源勾引魟 (ResourceGoingHome)

這是一個專為台灣垃圾回收與資源再利用設計的社群協調應用程式。透過 AI 分類技術，將產出資源的家戶（資小花）、協助運送的志工（資小蜂）與提供場地處理的機構（資小園）連結在一起。

## 名詞定義

1. 可回收物：可回收的資源，例如：寶特瓶、紙杯、紙箱、塑膠袋、玻璃瓶、鐵鋁罐……等等。

## 核心角色定義

本系統使用者可擁有多重身份，角色定義如下：

1. **資小花 (資源小野花)**
   - **定義**：一般使用者，可回收物的產出者。
   - **需求**：將垃圾拍照，由 AI 從照片中分類出可回收物，引導分類與前置處理，並送交回收。
   - **必要資訊**：所在地地址（用於計算鄰近性）。

2. **資小蜂 (資源小蜜蜂)**
   - **定義**：可回收物的收集者與運送者。
   - **必要資訊**：
     - 回收類別（願收受的可回收物種類）。
     - 前置處理要求（例如：需清洗、去標籤）。
     - 所在地地址。

3. **資小園 (資源小花園)**
   - **定義**：處理場地提供者。擁有堆置空間或處理設備。
   - **必要資訊**：
     - 回收類別（願收受的可回收物種類）。
     - 前置處理要求。
     - 場地收受地址（不提供到府收送）。

## 運作流程 (APP Workflow)

### 1. 拍照分類
資小花拍攝垃圾照片並透過 APP 上傳。

### 2. AI 智慧分析與媒合
AI 進行以下處理：
- **分類**：識別照片中的可回收物種類與數量。
- **媒合**：從系統中過濾出「距離鄰近」且「願意收受該類別」的資小蜂或資小園。
- **回傳建議**：
  - 分類與處理建議。
  - **結構化記錄列表**：包含種類、名稱、個數、符合條件的資小蜂/資小園清單、針對該項目的前置處理要求、以及所在地縣市法規建議。
- **資料儲存**：包含資小花 ID、地址、時間戳記與 AI 分析結果。

### 3. 前置處理與通知
資小花完成前置處理後，從清單中勾選一個資小蜂或資小園，發出通知。

### 4. 資小蜂回饋 (到府收送)
資小蜂收到通知後，可檢視記錄並透過留言與資小花溝通到府收受事宜。

### 5. 資小園回饋 (場地收受)
資小園收到通知後，可檢視記錄並留言溝通。APP 會特別提醒資小花：**資小園不提供到府服務，需自行將資源送往場地地址**。

## 版本里程碑 （Version Milestones）

### 版本 1.0
版本1.0須實作完成以下使用情境：
1. 匿名使用者選擇居住縣市、拍照垃圾，將照片 and 居住縣市上傳給AI，AI從照片中分類出可回收物，AI依據居住縣市給予在地的回收建議。
2. 資小花（註冊使用者）註冊時必須提供地址。資小花可在首頁登入，登入後可拍照垃圾，將照片和資小花的地址上傳給AI，AI從照片中分類出可回收物，AI依照資小花的地址給予在地的回收建議。

#### 頁面規劃與功能規格

| 頁面名稱 | 核心功能 | 功能規格說明 |
| :--- | :--- | :--- |
| **首頁 (Landing)** | 身份選擇與登入 | 1. 提供「開始拍照」鈕，點擊後若未登入即自動進行「匿名登入」。<br>2. 提供「Google 登入」按鈕供資小花註冊/登入。 |
| **設定頁 (Setup)** | 區域位置確認 | 1. **資小花路徑**：使用者必須輸入「詳細地址」（作為後續鄰近性計算基準）。<br>2. **匿名路徑**：使用者須選擇「居住縣市」（作為 AI 檢索法規基準）。<br>3. 資料將存儲於 local 與 Firebase 並導向主功能頁。 |
| **主頁/相機 (Home/Core)** | 觸發分類 | 1. 核心「拍照分類」大按鈕。<br>2. 顯示個人最近的回收紀錄（環保足跡）。<br>3. 頁面需有角標顯示當前角色身份與所在地址資。 |
| **分類結果 (Result)** | AI 分析與回報 | 1. 顯示分類出的可回收物清單（類別、個數）。<br>2. **在地化建議區**：AI 需根據註冊地址/縣市，生成具體的回收指引（如：垃圾車收受時間、特殊分類規定）。<br>3. 儲存分類結果至資料庫。 |
| **歷史紀錄 (History)** | 成果檢視 | 讓使用者查看過去所有拍攝過的照片與 AI 給出的回收足跡。 |
